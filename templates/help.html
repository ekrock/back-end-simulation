Quick Start

  1. Download the CSV template.
  2. Edit it to define your assembly line (stations, robot types, fleet size, and job parameters).
  3. Upload the CSV on the home page and click Run Simulation.
  4. Explore the results: OEE metrics, robot and station utilization, and a full tick-by-tick event log.
  5. Compare runs on the Cost vs. Performance chart.

CSV Format

The CSV is divided into named sections. Each section starts with a header row like [SIMULATION]. Blank lines between sections are ignored.

[SIMULATION]

FieldDescription
nameDisplay name for the simulation run.
descriptionOptional free-text description.
max_ticksHard stop — simulation ends when this tick count is reached, even if the job is not finished. Max: 50,000.

[JOB]

FieldDescription
nameName of the job (e.g. "Cable Back-End Assembly").
parts_to_buildNumber of parts to complete. Simulation ends early if all parts finish before max_ticks. Max: 1,000.
target_ticksPreferred cycle-time target per station action. Used by the robot assignment rule: the cheapest robot whose ticks_per_action is below this target is assigned first.

[LINE]

FieldDescription
input_buffer_sizeMax parts waiting to enter the first station.
output_buffer_sizeMax parts waiting in the output buffer after the last station.
central_store_distance_metersOne-way distance from the line to Central Store. AMRs travel at their speed (meters/tick) each way.
fetch_trigger_thresholdAMR fetches a batch from Central Store when the input buffer falls to this count.
deliver_trigger_thresholdAMR delivers to Central Store when the output buffer reaches this count.

[STATIONS]

One row per assembly station, in order. Header row station_name,action_name is required.

ColumnDescription
station_nameHuman-readable label shown in results (e.g. "Insert Cable into Port A").
action_nameInternal identifier matched to robot action times (e.g. "insert_cable_port_a"). No spaces.

Max: 20 stations.

[ROBOT_TYPES]

Each robot type uses two consecutive rows:

RowFormatDescription
1type_name, speed, costtype_name: display label. speed: meters per tick (leave blank for stationary robots). cost: fleet cost in dollars per unit.
2action_name, ticks, action_name, ticks, …Pairs of action name and ticks required. Must include all actions performed by this robot type. Use 0 for AMR fetch/deliver actions (handled by travel time).

Max: 10 robot types, 20 robots per type.

[ROBOTS]

Fleet size. Header row type_name,count is required.

ColumnDescription
type_nameMust match a type defined in [ROBOT_TYPES].
countNumber of robots of this type in the fleet.
